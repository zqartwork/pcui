CDN 與 Local Setup 架構

下載或使用CDN說明

  • 下載： 本地端開發時，為了避免網路問題而造成顯示問題，會使用下載的方式將套件下載並且放入專案資料夾中。缺點是第一次載入時間較長，但是之後載入速度幾乎接近0秒。
  • CDN： 節省下載資源，進入Production之後可以有效減少流量以及使用者第一次載入時間，之後載入時間相當。現階段企業廣用此方式載入套件。

常用外接套件

  1. meyer-reset.css 梅爾重置

    用以去除基礎style，減少每次設定系統style的功夫。

  2. animate.css + wow.js 滾輪進場動畫

    簡易 Scroll Animation，網站常用一次性進場動畫。

    安裝文件以及使用辦法：https://wowjs.uk/docs.html

  3. OwlCarousel 貓頭鷹輪播套件

    輕量級輪播套件，RWD、跨瀏覽器以及手勢滑動，Codepen 上也有很多改寫 Source 分享。

    安裝文件以及使用辦法：https://owlcarousel2.github.io/OwlCarousel2/

  4. FontAwesome.js 免費開源圖示庫

    此資源提供 Source Code, svg 圖檔，現今常用軟體縮圖 icon 也有提供。

    圖示搜尋表：https://fontawesome.com/icons?d=gallery&p=2

Setup 架構

Before we start to styling our website, preset some common rules. ( Customize and Suggestion )

在我們開始裝飾網站前，請先把規則訂好，輸入你的UI規則並且這邊會給您一些建議。這是我個人習慣的架構，可視情況做個人調整。


    MyWebsiteName/
    |--css/
    |  |--main.css // 主要compile出來的檔案
    |
    |--js/
    |  |--custom.js // Function Library 掛載在檔案的最下方，確保Loading得到
    |
    |--img/ // 推薦CDN化 - 後續會有補充 PicGo 免費圖床上載
    |  |--type/
    |
    |--scss/
    |  |--_pageName.scss // 非通用獨立專案的 scss，方便往後可以拔乾淨
    |  |--_kit.scss // UI KIT Library 以及 預寫的 extend, mixin 物件
    |  |--_style.scss // 全站通用的 Style
    |  |--_rwd.scss // 全站通用的 RWD
    |  |--main.scss // Import compile 主要檔案
    |
    |--index.html
    |--pageName.html
    |--pageName.html

main.scss 內部設定：( 對 scss 不熟悉的朋友可以前往： 【SASS全秘笈】 ) 以下是我固定的全站設定。


@import "kit"; /*越下方層級越高*/

/*全站基本設定*/
html, body, * {
    box-sizing: border-box;
    font-family: $font;
    position: relative;
    cursor: default;
    line-height: 1.5;
}

body {
    font-size: 10px;
    min-height: 100vh; //Sticky Footer
}

@import "style";
@import "rwd";
@import "singlepage";

_kit.scss 內部設定：字體可以外部掛載或者 Local 掛載，後者建議轉 CDN 外部掛載，避免 IE 死去。缺點是會比較慢 Loading 進網站，建議使用 Loader 阻擋一下第一次載入破字；
此份 scss 主要用來將 Figma 或 UI 設計軟體上 Library 的 Asset 輸入，方便之後撰寫取用，component style 也可以預先寫好變成 mixin!


/*
